# Standard FE-003 — Moderation Status Label Standard

This standard defines the canonical approach for displaying moderation status labels and badges in RealtorNet.

## Principle

**Use the centralized moderation label map, never hardcode status text.**

All moderation status display must use the `moderationStatusLabel` map from `src/features/properties/lib/moderation.ts`.

## Moderation Status Constants

### Status Enum
```tsx
export const MODERATION_STATUS = {
  draft: "draft",
  agencyReview: "agency_review",
  agencyRejected: "agency_rejected",
  adminReview: "admin_review",
  adminRejected: "admin_rejected",
  live: "live",
  revoked: "revoked",
  pendingReview: "pending_review",
  agencyApproved: "agency_approved",
  verified: "verified",
  rejected: "rejected",
} as const satisfies Record<string, ModerationStatus>;
```

### Label Map

export const moderationStatusLabel: Record<ModerationStatus, string> = {

## Usage Patterns

### Import the Constants
```tsx
import {
  MODERATION_STATUS,
  moderationStatusLabel,
  moderationStatusBadgeVariant,
} from "@/features/properties/lib/moderation";
```

### Display Status Label
```tsx
// ✅ Correct: Use the label map
<Badge variant={moderationStatusBadgeVariant[property.moderation_status]}>
  {moderationStatusLabel[property.moderation_status]}
</Badge>
```

### Display Status with Context
```tsx
// ✅ Correct: Use label map with additional context
<div className="flex items-center gap-2">
  <Badge variant={moderationStatusBadgeVariant[property.moderation_status]}>
    {moderationStatusLabel[property.moderation_status]}
  </Badge>
  {property.moderation_status === MODERATION_STATUS.revoked && (
    <span className="text-sm text-gray-500">
      Awaiting agency instruction
    </span>
  )}
</div>
```

### Conditional Rendering Based on Status
```tsx
// ✅ Correct: Use constants for comparison
{property.moderation_status === MODERATION_STATUS.live && (
  <div className="text-green-600">Listing is live</div>
)}
```

## Helper Functions

### Check if Verified
```tsx
export function isVerifiedModerationStatus(status: ModerationStatus) {
  return status === MODERATION_STATUS.live || status === MODERATION_STATUS.verified;
}
```

Usage:
```tsx
{isVerifiedModerationStatus(property.moderation_status) && (
  <VerifiedBadge />
)}
```

### Check if Reason Should Be Shown
```tsx
export function shouldShowModerationReason(status: ModerationStatus) {
  return (
    status === MODERATION_STATUS.agencyRejected ||
    status === MODERATION_STATUS.adminRejected ||
    status === MODERATION_STATUS.rejected ||
    status === MODERATION_STATUS.revoked
  );
}
```

Usage:
```tsx
{shouldShowModerationReason(property.moderation_status) && property.rejection_reason && (
  <div className="rounded-lg bg-red-50 p-3 text-sm text-red-700">
    {property.rejection_reason}
  </div>
)}
```

### Check if Agency Name Should Be Shown
```tsx
export const AGENCY_NAME_STATES: ModerationStatus[] = [
  "admin_review",
];

export const shouldShowAgencyName = (status: ModerationStatus) =>
  AGENCY_NAME_STATES.includes(status);
```

Usage:
```tsx
{shouldShowAgencyName(property.moderation_status) && (
  <span>{property.agency_name}</span>
)}
```

## CTA Descriptors

### Admin Revocation CTA
```tsx
export const getAdminRevocationCta = (
  status: ModerationStatus,
  hasInstruction: boolean | null | undefined,
): CtaDescriptor => {
  if (status === 'revoked' && !hasInstruction) return { label: 'Awaiting agency instruction', action: null }
  if (status === 'revoked' && hasInstruction) return { label: 'Awaiting agent action', action: null }
  if (['draft', 'agency_review', 'admin_review'].includes(status)) return { label: 'In progress', action: null }
  if (status === 'live') return { label: 'Restored', action: null }
  if (status === 'admin_rejected') return { label: 'Rejected', action: null }
  return { label: status, action: null }
}
```

### Admin Rejection CTA
```tsx
export const getAdminRejectionCta = (
  status: ModerationStatus,
  hasInstruction: boolean | null | undefined,
): CtaDescriptor => {
  if (status === 'admin_rejected' && !hasInstruction) return { label: 'Reinstate', action: 'reinstate' }
  if (status === 'admin_rejected' && hasInstruction) return { label: 'Awaiting agent action', action: null }
  if (['draft', 'agency_review', 'admin_review'].includes(status)) return { label: 'In progress', action: null }
  if (status === 'live') return { label: 'Resolved — listing live', action: null }
  if (status === 'revoked') return { label: 'Revoked', action: null }
  return { label: status, action: null }
}
```

## Anti-Patterns

### Hardcoded Status Text
```tsx
// ❌ Incorrect: Hardcoded status text
<Badge>Agency review</Badge>
```

### Hardcoded Status Comparison
```tsx
// ❌ Incorrect: Hardcoded status string
{property.moderation_status === "agency_review" && (
  <div>In review</div>
)}
```

### Missing Badge Variant
```tsx
// ❌ Incorrect: Not using badge variant map
<Badge>{property.moderation_status}</Badge>
```

### Manual Status Mapping
```tsx
// ❌ Incorrect: Manual status mapping instead of using centralized map
const statusLabels = {
  draft: "Draft",
  agency_review: "Agency review",
  // ...
};
<Badge>{statusLabels[property.moderation_status]}</Badge>
```

## Correct Patterns

### Using Centralized Map
```tsx
// ✅ Correct: Use centralized label map
import { moderationStatusLabel, moderationStatusBadgeVariant } from "@/features/properties/lib/moderation";

<Badge variant={moderationStatusBadgeVariant[property.moderation_status]}>
  {moderationStatusLabel[property.moderation_status]}
</Badge>
```

### Using Constants for Comparison
```tsx
// ✅ Correct: Use constants for comparison
import { MODERATION_STATUS } from "@/features/properties/lib/moderation";

{property.moderation_status === MODERATION_STATUS.agencyReview && (
  <div>In review</div>
)}
```

### Using Helper Functions
```tsx
// ✅ Correct: Use helper functions
import { isVerifiedModerationStatus, shouldShowModerationReason } from "@/features/properties/lib/moderation";

{isVerifiedModerationStatus(property.moderation_status) && (
  <VerifiedBadge />
)}

{shouldShowModerationReason(property.moderation_status) && property.rejection_reason && (
  <div className="rounded-lg bg-red-50 p-3 text-sm text-red-700">
    {property.rejection_reason}
  </div>
)}
```

## Migration Guide

When updating existing moderation status display:

1. Import the constants and helpers from `@/features/properties/lib/moderation`
2. Replace hardcoded status strings with `MODERATION_STATUS` constants
3. Replace hardcoded status labels with `moderationStatusLabel` map
4. Replace hardcoded badge variants with `moderationStatusBadgeVariant` map
5. Use helper functions for common status checks
6. Remove any local status label mappings

## Adding New Statuses

When adding a new moderation status:

1. Add the status to `MODERATION_STATUS` constant
2. Add the label to `moderationStatusLabel` map
3. Add the badge variant to `moderationStatusBadgeVariant` map
4. Update helper functions if the new status affects their logic
5. Update CTA descriptors if the new status affects admin workflows

## Testing

Verify moderation status display by:

1. Checking all status values display correct labels
2. Checking badge variants match the status semantic meaning
3. Checking helper functions return correct boolean values
4. Checking conditional rendering works for all statuses
5. Checking CTA descriptors display correct labels and actions
6. Checking legacy statuses display correctly with "Legacy" prefix

## Badge Variant Semantics

- **success**: Positive states (live, verified)
- **warning**: In-progress or pending states (agency_review, admin_review, pending_review, agency_approved)
- **danger**: Negative states (agency_rejected, admin_rejected, rejected, revoked)
- **default**: Neutral states (draft)
