# Standard UI-005 — Overlay Architecture Pattern

This is RealtorNet's canonical overlay architecture.

```
Button

↓

Overlay Controller

↓

Positioning Engine

↓

Portal

↓

document.body
```

Responsibilities are separated:

* Trigger manages interaction.
* Controller manages open/close state.
* Positioning engine computes placement.
* Portal manages rendering.
* Overlay renders content.

This pattern scales from one dropdown to hundreds.

## Architecture Components

### 1. Trigger
The trigger is the user-facing element that initiates the overlay.

**Responsibilities:**
- Handle user interaction (click, focus, hover)
- Manage trigger-specific state (disabled, loading)
- Provide visual feedback

**Example:**
```tsx
function DropdownTrigger({ children, onClick }) {
  return (
    <button 
      type="button"
      onClick={onClick}
      className="inline-flex items-center"
    >
      {children}
    </button>
  );
}
```

### 2. Overlay Controller
The controller manages the overlay's open/close state and lifecycle.

**Responsibilities:**
- Manage open/close state
- Handle click-outside dismissal
- Handle keyboard dismissal (Escape)
- Manage focus trapping (for modals)
- Coordinate with trigger

**Example:**
```tsx
function useOverlayController({ initialOpen = false }) {
  const [isOpen, setIsOpen] = useState(initialOpen);
  const triggerRef = useRef<HTMLButtonElement>(null);

  const open = useCallback(() => setIsOpen(true), []);
  const close = useCallback(() => setIsOpen(false), []);
  const toggle = useCallback(() => setIsOpen(v => !v), []);

  useEffect(() => {
    if (!isOpen) return;

    const handleClickOutside = (event: MouseEvent) => {
      if (triggerRef.current?.contains(event.target as Node)) return;
      close();
    };

    const handleEscape = (event: KeyboardEvent) => {
      if (event.key === 'Escape') close();
    };

    document.addEventListener('mousedown', handleClickOutside);
    document.addEventListener('keydown', handleEscape);

    return () => {
      document.removeEventListener('mousedown', handleClickOutside);
      document.removeEventListener('keydown', handleEscape);
    };
  }, [isOpen, close]);

  return { isOpen, open, close, toggle, triggerRef };
}
```

### 3. Positioning Engine
The positioning engine computes where the overlay should render.

**Responsibilities:**
- Calculate overlay position relative to trigger
- Handle viewport boundaries
- Handle scroll position
- Update position on resize/scroll
- Support multiple alignment options

**Example:**
```tsx
function useOverlayPosition(triggerRef, { align = 'start', offset = 4 }) {
  const [position, setPosition] = useState({ top: 0, left: 0, width: 0 });

  const updatePosition = useCallback(() => {
    const trigger = triggerRef.current;
    if (!trigger) return;

    const rect = trigger.getBoundingClientRect();
    const scrollY = window.scrollY || window.pageYOffset;
    const scrollX = window.scrollX || window.pageXOffset;

    let left = rect.left;
    let width = rect.width;

    if (align === 'end') {
      left = rect.right - width;
    } else if (align === 'center') {
      left = rect.left + rect.width / 2 - width / 2;
    }

    setPosition({
      top: rect.bottom + scrollY + offset,
      left: left + scrollX,
      width: Math.max(width, 200),
    });
  }, [triggerRef, align, offset]);

  useEffect(() => {
    updatePosition();
    window.addEventListener('resize', updatePosition);
    window.addEventListener('scroll', updatePosition);

    return () => {
      window.removeEventListener('resize', updatePosition);
      window.removeEventListener('scroll', updatePosition);
    };
  }, [updatePosition]);

  return { position, updatePosition };
}
```

### 4. Portal
The portal manages rendering the overlay outside the normal DOM tree.

**Responsibilities:**
- Render overlay to `document.body`
- Escape clipping contexts
- Escape stacking contexts
- Maintain accessibility (focus management)

**Example:**
```tsx
import { createPortal } from 'react-dom';

function OverlayPortal({ children }) {
  if (typeof window === 'undefined') return null;
  return createPortal(children, document.body);
}
```

### 5. Overlay
The overlay is the actual content that appears above the page.

**Responsibilities:**
- Render overlay content
- Apply correct z-index (per UI-002)
- Handle accessibility (ARIA attributes)
- Support keyboard navigation

**Example:**
```tsx
function Overlay({ children, position, className }) {
  return (
    <div
      className={`fixed z-50 ${className}`}
      style={{
        top: `${position.top}px`,
        left: `${position.left}px`,
        width: `${position.width}px`,
      }}
      role="dialog"
    >
      {children}
    </div>
  );
}
```

## Complete Implementation

### Dropdown Component
```tsx
import { createPortal } from 'react-dom';
import { useState, useRef, useEffect, useCallback } from 'react';

function Dropdown({ trigger, children, align = 'start' }) {
  const [isOpen, setIsOpen] = useState(false);
  const triggerRef = useRef<HTMLButtonElement>(null);
  const [position, setPosition] = useState({ top: 0, left: 0, width: 0 });

  // Controller
  const open = useCallback(() => setIsOpen(true), []);
  const close = useCallback(() => setIsOpen(false), []);
  const toggle = useCallback(() => setIsOpen(v => !v), []);

  // Positioning Engine
  useEffect(() => {
    if (!isOpen || !triggerRef.current) return;

    const updatePosition = () => {
      const rect = triggerRef.current!.getBoundingClientRect();
      const scrollY = window.scrollY || window.pageYOffset;
      const scrollX = window.scrollX || window.pageXOffset;

      let left = rect.left;
      let width = rect.width;

      if (align === 'end') {
        left = rect.right - width;
      } else if (align === 'center') {
        left = rect.left + rect.width / 2 - width / 2;
      }

      setPosition({
        top: rect.bottom + scrollY + 4,
        left: left + scrollX,
        width: Math.max(width, 200),
      });
    };

    updatePosition();
    window.addEventListener('resize', updatePosition);
    window.addEventListener('scroll', updatePosition);

    return () => {
      window.removeEventListener('resize', updatePosition);
      window.removeEventListener('scroll', updatePosition);
    };
  }, [isOpen, align]);

  // Click-outside handling
  useEffect(() => {
    if (!isOpen) return;

    const handleClickOutside = (event: MouseEvent) => {
      if (triggerRef.current?.contains(event.target as Node)) return;
      close();
    };

    const handleEscape = (event: KeyboardEvent) => {
      if (event.key === 'Escape') close();
    };

    document.addEventListener('mousedown', handleClickOutside);
    document.addEventListener('keydown', handleEscape);

    return () => {
      document.removeEventListener('mousedown', handleClickOutside);
      document.removeEventListener('keydown', handleEscape);
    };
  }, [isOpen, close]);

  // Portal + Overlay
  const overlay = isOpen && typeof window !== 'undefined' ? (
    createPortal(
      <div
        className="fixed z-50 rounded-lg border border-gray-200 bg-white shadow-lg"
        style={{
          top: `${position.top}px`,
          left: `${position.left}px`,
          width: `${position.width}px`,
        }}
        role="dialog"
      >
        {children}
      </div>,
      document.body
    )
  ) : null;

  return (
    <>
      {trigger({ ref: triggerRef, onClick: toggle, isOpen })}
      {overlay}
    </>
  );
}
```

### Usage
```tsx
<Dropdown
  trigger={({ ref, onClick, isOpen }) => (
    <button
      ref={ref}
      onClick={onClick}
      className="inline-flex items-center gap-2"
    >
      Select <ChevronDown className={isOpen ? 'rotate-180' : ''} />
    </button>
  )}
>
  <ul>
    <li>Option 1</li>
    <li>Option 2</li>
    <li>Option 3</li>
  </ul>
</Dropdown>
```

## Scaling Considerations

### Single Dropdown
- Manual implementation is acceptable
- Custom positioning logic per component
- Good for one-off use cases

### Multiple Dropdowns
- Extract shared controller logic
- Extract shared positioning logic
- Create reusable `useOverlay` hook

### Many Overlays
- Create shared overlay component library
- Standardize on Radix UI or similar
- Maintain consistent behavior across app

## RealtorNet Implementation

### Current State
- Hero search dropdown uses manual portal implementation
- Property card filters use inline positioning
- Modals use portal rendering
- Toasts use portal rendering

### Migration Path
1. Extract shared `useOverlay` hook
2. Standardize on Radix UI primitives where possible
3. Apply this architecture to new overlay components
4. Gradually migrate existing overlays to shared pattern

### Anti-Patterns to Avoid

#### Inline State in Component
```tsx
// ❌ Incorrect: State and positioning mixed in component
function BadDropdown() {
  const [isOpen, setIsOpen] = useState(false);
  const [position, setPosition] = useState({ top: 0, left: 0 });
  
  // All logic mixed together
  useEffect(() => {
    // Positioning logic
    // Controller logic
    // Click handling
  }, [isOpen]);
  
  return (
    <>
      <button onClick={() => setIsOpen(!isOpen)}>Toggle</button>
      {isOpen && createPortal(/* ... */, document.body)}
    </>
  );
}
```

#### Missing Separation
```tsx
// ❌ Incorrect: No separation of concerns
function BadOverlay({ trigger, content }) {
  return (
    <>
      {trigger}
      {content && createPortal(content, document.body)}
    </>
  );
}
```

### Correct Patterns

#### Separated Concerns
```tsx
// ✅ Correct: Clear separation of concerns
const { isOpen, open, close, triggerRef } = useOverlayController();
const { position } = useOverlayPosition(triggerRef, { align: 'start' });

return (
  <>
    <Trigger ref={triggerRef} onClick={toggle} />
    {isOpen && <OverlayPortal><Overlay position={position} /></OverlayPortal>}
  </>
);
```

#### Reusable Hook
```tsx
// ✅ Correct: Extracted reusable logic
function useOverlay({ align = 'start' }) {
  const controller = useOverlayController();
  const position = useOverlayPosition(controller.triggerRef, { align });
  return { ...controller, position };
}
```

## Benefits

1. **Consistency**: All overlays behave the same way
2. **Maintainability**: Clear separation of concerns
3. **Testability**: Each component can be tested independently
4. **Scalability**: Pattern works for one or hundreds of overlays
5. **Accessibility**: Centralized ARIA and focus management
