# Standard UI-004 — Component Rendering Decision Matrix

Every new component answers one question first:

> **Where should this component live in the DOM?**

## Decision Matrix

| Component               | Rendering Strategy | Rationale                          |
| ----------------------- | ------------------ | ---------------------------------- |
| Card                    | Inline             | Static content, no overflow risk  |
| Button                  | Inline             | Static content, no overflow risk  |
| Modal                   | Portal             | Must escape all stacking contexts |
| Dropdown (inside table) | Inline             | Container is safe, scrolls with parent |
| Hero search dropdown    | Portal             | Must escape hero overflow clipping |
| Tooltip                 | Portal             | Must escape clipping, positioning flexibility |
| Toast                   | Portal             | Global positioning, above all content |
| Context menu            | Portal             | Must escape all contexts, viewport positioning |

## When to Use Inline Rendering

Use inline rendering when:

1. The component is contained within a safe, unclipped container
2. The component should scroll with its parent
3. The component is part of a composite component (e.g., within a table cell)
4. The component's positioning is relative to its parent
5. The component doesn't need to escape stacking contexts

### Examples

#### Card Component
```tsx
// ✅ Correct: Card is static content
function Card({ children }) {
  return (
    <div className="rounded-lg border bg-white p-4">
      {children}
    </div>
  );
}
```

#### Table Dropdown
```tsx
// ✅ Correct: Dropdown inside safe table container
function TableDropdown({ row }) {
  return (
    <div className="relative">
      <button>Menu</button>
      {isOpen && (
        <div className="absolute top-full left-0 mt-2">
          {/* Dropdown scrolls with table */}
        </div>
      )}
    </div>
  );
}
```

## When to Use Portal Rendering

Use portal rendering when:

1. The component needs to extend beyond a clipped container (`overflow: hidden`)
2. The component needs to escape a stacking context that limits z-index
3. The component needs to remain visible while scrolling
4. The component should not be affected by parent transforms or filters
5. The component needs global positioning (viewport-relative)

### Examples

#### Modal
```tsx
// ✅ Correct: Modal must escape all contexts
import { createPortal } from 'react-dom';

function Modal({ children }) {
  return createPortal(
    <div className="fixed inset-0 z-50">
      {children}
    </div>,
    document.body
  );
}
```

#### Hero Search Dropdown
```tsx
// ✅ Correct: Dropdown must escape hero overflow clipping
import { createPortal } from 'react-dom';

function SearchDropdown({ suggestions, position }) {
  return createPortal(
    <ul 
      className="fixed z-50"
      style={{ top: `${position.top}px`, left: `${position.left}px` }}
    >
      {suggestions.map(/* ... */)}
    </ul>,
    document.body
  );
}
```

#### Toast
```tsx
// ✅ Correct: Toast needs global positioning
import { createPortal } from 'react-dom';

function Toast({ message }) {
  return createPortal(
    <div className="fixed bottom-4 right-4 z-60">
      {message}
    </div>,
    document.body
  );
}
```

## Decision Flowchart

```
Is the component an overlay?
│
├─ No → Use inline rendering
│
└─ Yes → Will it overflow its container?
    │
    ├─ No → Can it escape stacking contexts inline?
    │   │
    │   ├─ Yes → Use inline rendering
    │   │
    │   └─ No → Use portal rendering
    │
    └─ Yes → Use portal rendering
```

## Implementation Checklist

Before implementing a new component:

- [ ] Does this component need to render above other content?
- [ ] Is the parent container clipped (`overflow: hidden`)?
- [ ] Is the parent container a stacking context?
- [ ] Should the component scroll with its parent?
- [ ] Should the component remain visible during scroll?
- [ ] Does the component need viewport-relative positioning?

## RealtorNet Patterns

### Inline Components
- Property cards
- Agency cards
- Agent cards
- Buttons, inputs, form fields
- Table cell dropdowns
- Filter dropdowns in safe containers

### Portal Components
- Hero search dropdown
- Modals and dialogs
- Toasts and notifications
- Navigation drawers
- Context menus
- Tooltips

### Hybrid Approach
Some components support both strategies based on context:

```tsx
function Dropdown({ children, usePortal = false }) {
  if (usePortal) {
    return createPortal(/* ... */, document.body);
  }
  return <div className="absolute">{children}</div>;
}
```

## Migration Guide

When converting an inline component to portal:

1. Add `import { createPortal } from 'react-dom'`
2. Calculate position using `getBoundingClientRect()`
3. Add scroll offset (`window.scrollY`, `window.scrollX`)
4. Replace `absolute` with `fixed` positioning
5. Render to `document.body`
6. Update z-index according to UI-002 scale
7. Test across viewports and scroll positions

## Anti-Patterns

### Unnecessary Portals
```tsx
// ❌ Incorrect: Portal for static content
function Card({ children }) {
  return createPortal(
    <div className="rounded-lg border">{children}</div>,
    document.body
  );
}
```

### Missing Position Calculation
```tsx
// ❌ Incorrect: Portal without position calculation
function Dropdown({ children }) {
  return createPortal(
    <div className="fixed">{children}</div>,
    document.body
  );
}
```

### Mixing Strategies Without Reason
```tsx
// ❌ Incorrect: Arbitrary strategy selection
function Component({ usePortal }) {
  return usePortal ? portalRender() : inlineRender();
}
```

## Correct Patterns

### Context-Aware Rendering
```tsx
// ✅ Correct: Strategy based on container safety
function Dropdown({ containerRef, children }) {
  const container = containerRef.current;
  const isClipped = getComputedStyle(container).overflow === 'hidden';
  
  if (isClipped) {
    return createPortal(portalContent, document.body);
  }
  return <div className="absolute">{children}</div>;
}
```

### Documented Decision
```tsx
// ✅ Correct: Document why portal is used
// Portal used to escape hero overflow: hidden
// See UI-001 for decision matrix
function HeroSearchDropdown({ children }) {
  return createPortal(/* ... */, document.body);
}
```
