# Standard UI-001 — Overlay & Floating Layer Standard

**Purpose**

Defines the canonical implementation for every UI element that floats above normal document flow.

Applies to:

* Dropdowns
* Popovers
* Menus
* Tooltips
* Date pickers
* Autocomplete
* Command palettes
* Context menus

## Decision Matrix

| Situation                                         | Standard                        |
| ------------------------------------------------- | ------------------------------- |
| Overlay remains inside its container              | `relative` + `absolute`         |
| Overlay may overflow parent                       | Portal                          |
| Overlay must escape clipping or stacking contexts | Portal                          |
| Overlay tracks viewport instead of element        | `fixed` with positioning engine |

## Rule

Never solve overlay visibility problems by continually increasing `z-index`.

## Implementation Examples

### Inline Dropdown (Safe Container)
```tsx
// ✅ Correct: Dropdown stays within safe container
<div className="relative">
  <button>Toggle</button>
  {isOpen && (
    <div className="absolute top-full left-0 mt-2">
      {/* Dropdown content */}
    </div>
  )}
</div>
```

### Portal Dropdown (Escaping Clipping)
```tsx
// ✅ Correct: Portal escapes overflow clipping
import { createPortal } from 'react-dom';

const { top, left, width } = inputRef.current?.getBoundingClientRect() ?? {};

return createPortal(
  <ul 
    className="fixed z-50"
    style={{ top: `${top + height}px`, left: `${left}px`, width: `${width}px` }}
  >
    {/* Dropdown content */}
  </ul>,
  document.body
);
```

### Anti-Pattern: Z-Index Escalation
```tsx
// ❌ Incorrect: Never solve clipping with z-index
<div className="relative overflow-hidden">
  <button>Toggle</button>
  {isOpen && (
    <div className="absolute z-[9999]">
      {/* This won't escape overflow: hidden */}
    </div>
  )}
</div>
```

## When to Use Portals

Use portals when:

1. The overlay needs to extend beyond a clipped container (`overflow: hidden`)
2. The overlay needs to escape a stacking context that limits z-index
3. The overlay needs to remain visible while scrolling
4. The overlay should not be affected by parent transforms or filters

Use inline positioning when:

1. The overlay is contained within a safe, unclipped container
2. The overlay should scroll with its parent
3. The overlay is part of a composite component (e.g., within a table cell)

## RealtorNet Specifics

- Hero search dropdowns use portal rendering to escape the hero's `overflow-hidden`
- Filter dropdowns within tables use inline positioning
- Modals and drawers always use portal rendering
- Toasts use portal rendering with a fixed position
