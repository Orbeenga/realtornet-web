# Standard UI-003 — Runtime Debugging Standard

Every UI bug follows the same sequence.

```
Reproduce

↓

Identify runtime owner

↓

Inspect DOM

↓

Inspect computed styles

↓

Inspect stacking contexts

↓

Inspect overflow

↓

Inspect network/state

↓

Prove root cause

↓

Implement

↓

Verify runtime behaviour
```

No implementation begins until the root cause has evidence.

## Step-by-Step Process

### 1. Reproduce
- Open the page in a browser
- Trigger the issue consistently
- Note the exact steps to reproduce
- Capture screenshots or screen recordings if visual

### 2. Identify Runtime Owner
- Which component owns the problematic behavior?
- Which hook or state drives the rendering?
- Which user interaction triggers the issue?
- Check component hierarchy in React DevTools

### 3. Inspect DOM
- Open DevTools Elements panel
- Locate the problematic element
- Check if the element exists in the DOM
- Verify the element's position in the DOM tree
- Check for conditional rendering (`display: none`, missing from DOM)

### 4. Inspect Computed Styles
- Check Computed tab in DevTools
- Verify `position`, `top`, `left`, `width`, `height`
- Check `z-index` value
- Check `display`, `visibility`, `opacity`
- Check `transform`, `filter`, `perspective` (stacking context creators)

### 5. Inspect Stacking Contexts
- Walk up the ancestor chain
- Check each ancestor for stacking context creation:
  - `position: relative/absolute/fixed` + `z-index`
  - `opacity < 1`
  - `transform`, `filter`, `perspective`
  - `will-change: transform/opacity`
- Identify the nearest stacking context that constrains the element

### 6. Inspect Overflow
- Check each ancestor for `overflow: hidden`
- Check each ancestor for `overflow-y: hidden`
- Check each ancestor for `overflow-x: hidden`
- Identify the clipping container

### 7. Inspect Network/State
- Check network requests in DevTools Network tab
- Verify API responses are correct
- Check React state in DevTools
- Verify props and state are as expected
- Check for race conditions or stale state

### 8. Prove Root Cause
- Isolate the root cause with a minimal test
- Use temporary CSS changes to confirm (e.g., `overflow: visible`)
- Use console logging to verify state changes
- Document the evidence before implementing

### 9. Implement
- Apply the fix based on proven root cause
- Prefer architectural fixes (portals) over CSS workarounds
- Follow project standards (UI-001, UI-002)
- Make minimal, targeted changes

### 10. Verify Runtime Behaviour
- Test the fix in the same reproduction scenario
- Verify the original issue is resolved
- Check for regressions in related functionality
- Test across browsers and viewports
- Verify keyboard navigation and accessibility

## Diagnostic Tools

### DevTools Console
```js
// Check overflow on element and ancestors
function checkOverflow(element) {
  let current = element;
  while (current) {
    const style = getComputedStyle(current);
    console.log(current.className, {
      overflow: style.overflow,
      overflowY: style.overflowY,
      overflowX: style.overflowX,
    });
    current = current.parentElement;
  }
}

// Check stacking context on element and ancestors
function checkStackingContext(element) {
  let current = element;
  while (current) {
    const style = getComputedStyle(current);
    console.log(current.className, {
      position: style.position,
      zIndex: style.zIndex,
      opacity: style.opacity,
      transform: style.transform,
      filter: style.filter,
    });
    current = current.parentElement;
  }
}
```

### Temporary Diagnostic CSS
```js
// Temporarily disable overflow to confirm clipping
document.querySelector('.hero-container').style.overflow = 'visible';

// Temporarily elevate z-index to confirm stacking
document.querySelector('.dropdown').style.zIndex = '9999';
```

### React DevTools
- Use Components tab to inspect component hierarchy
- Use Profiler to identify performance issues
- Use Hooks tab to inspect state and props
- Use Settings to enable debug mode

## Common Patterns

### Clipping Diagnosis
1. Element is visible in DOM but not on screen
2. `overflow: hidden` on ancestor
3. Fix: Portal rendering or remove overflow

### Stacking Context Diagnosis
1. Element is rendered but covered by another element
2. Higher z-index doesn't work
3. Fix: Portal rendering or restructure DOM

### State Diagnosis
1. Element renders incorrectly based on state
2. State is stale or missing
3. Fix: Correct state management or cache invalidation

### Network Diagnosis
1. Element depends on API data
2. API response is incorrect or missing
3. Fix: Correct API call or error handling

## Anti-Patterns

### Implementing Without Diagnosis
```tsx
// ❌ Incorrect: Adding z-index without checking overflow
<div className="z-[9999]">Dropdown</div>
```

### Assuming Without Evidence
```tsx
// ❌ Incorrect: Assuming the issue is z-index
// "I'll just increase z-index"
```

### Skipping Verification
```tsx
// ❌ Incorrect: Implementing without testing
// "That should fix it"
```

## Correct Patterns

### Evidence-Based Implementation
```tsx
// ✅ Correct: Prove root cause first
// Console shows overflow: hidden on hero-container
// Fix: Use portal rendering
```

### Diagnostic Documentation
```tsx
// ✅ Correct: Document diagnostic steps
// 1. Confirmed element exists in DOM
// 2. Computed styles show correct positioning
// 3. Ancestor with overflow: hidden identified
// 4. Temporary overflow: visible confirms clipping
// 5. Portal rendering implemented
```

### Verification Checklist
```tsx
// ✅ Correct: Verify after implementation
// - Original issue resolved
// - No regressions
// - Mobile behavior verified
// - Accessibility verified
```
