# Standard UI-006 — UI Verification Checklist

No UI issue is closed until all applicable items pass.

## Minimum Acceptance Criteria

```
☐ Reproduced

☐ Runtime owner identified

☐ Root cause proven

☐ Fix implemented

☐ Original issue no longer reproducible

☐ No regressions observed

☐ Keyboard navigation verified

☐ Mobile behaviour verified

☐ Accessibility unaffected

☐ Responsive layout verified

☐ Browser refresh verified

☐ State persistence verified
```

## Detailed Verification Steps

### 1. Reproduced
- [ ] Open the page in a browser
- [ ] Trigger the issue using the reported steps
- [ ] Confirm the issue exists
- [ ] Document the exact reproduction steps
- [ ] Capture screenshots or screen recordings if visual

### 2. Runtime Owner Identified
- [ ] Identify which component owns the problematic behavior
- [ ] Identify which hook or state drives the rendering
- [ ] Identify which user interaction triggers the issue
- [ ] Verify component hierarchy in React DevTools
- [ ] Document the runtime owner in the issue description

### 3. Root Cause Proven
- [ ] Follow UI-003 debugging sequence
- [ ] Inspect DOM and confirm element existence
- [ ] Inspect computed styles and verify positioning
- [ ] Inspect stacking contexts and identify constraints
- [ ] Inspect overflow and identify clipping containers
- [ ] Inspect network/state and verify data flow
- [ ] Document the proven root cause with evidence

### 4. Fix Implemented
- [ ] Implement the fix based on proven root cause
- [ ] Follow project standards (UI-001, UI-002, UI-004, UI-005)
- [ ] Make minimal, targeted changes
- [ ] Document the implementation approach
- [ ] Ensure code follows project conventions

### 5. Original Issue No Longer Reproducible
- [ ] Test the fix in the same reproduction scenario
- [ ] Verify the original issue is resolved
- [ ] Test edge cases related to the fix
- [ ] Test across different viewports
- [ ] Test across different browsers (Chrome, Firefox, Safari)

### 6. No Regressions Observed
- [ ] Test related functionality that might be affected
- [ ] Test parent components
- [ ] Test child components
- [ ] Test sibling components
- [ ] Verify no new issues are introduced

### 7. Keyboard Navigation Verified
- [ ] Test Tab navigation through the component
- [ ] Test Enter/Space for activation
- [ ] Test Escape for dismissal (modals, dropdowns)
- [ ] Test Arrow keys for navigation (dropdowns, menus)
- [ ] Verify focus management (focus trapping for modals)
- [ ] Verify ARIA attributes are correct

### 8. Mobile Behaviour Verified
- [ ] Test on mobile viewport (375px width)
- [ ] Test on tablet viewport (768px width)
- [ ] Test touch interactions (tap, swipe)
- [ ] Test responsive layout behavior
- [ ] Test scroll behavior on mobile
- [ ] Verify mobile-specific issues are resolved

### 9. Accessibility Unaffected
- [ ] Verify screen reader compatibility
- [ ] Verify high contrast mode support
- [ ] Verify text scaling support
- [ ] Verify reduced motion support
- [ ] Verify color contrast ratios
- [ ] Verify ARIA labels and roles

### 10. Responsive Layout Verified
- [ ] Test on small screens (320px)
- [ ] Test on medium screens (768px)
- [ ] Test on large screens (1024px)
- [ ] Test on extra-large screens (1440px)
- [ ] Verify layout doesn't break at breakpoints
- [ ] Verify horizontal scrolling doesn't appear unexpectedly

### 11. Browser Refresh Verified
- [ ] Hard refresh the page (Ctrl+Shift+R / Cmd+Shift+R)
- [ ] Verify the fix persists after refresh
- [ ] Verify no hydration errors
- [ ] Verify no console errors
- [ ] Verify no warnings in DevTools

### 12. State Persistence Verified
- [ ] Test state survives page navigation
- [ ] Test state survives browser refresh
- [ ] Test state survives component unmount/remount
- [ ] Verify localStorage/sessionStorage if used
- [ ] Verify URL state if used
- [ ] Verify cache invalidation if used

## Context-Specific Additions

### Overlays and Dropdowns
- [ ] Verify dropdown opens above trigger
- [ ] Verify dropdown closes on click outside
- [ ] Verify dropdown closes on Escape
- [ ] Verify dropdown doesn't get clipped
- [ ] Verify dropdown z-index is correct

### Modals and Dialogs
- [ ] Verify modal opens with backdrop
- [ ] Verify modal traps focus
- [ ] Verify modal closes on backdrop click
- [ ] Verify modal closes on Escape
- [ ] Verify modal restores focus on close

### Forms and Inputs
- [ ] Verify form validation works
- [ ] Verify error messages display correctly
- [ ] Verify submit behavior is correct
- [ ] Verify form reset works
- [ ] Verify disabled state works

### Tables and Lists
- [ ] Verify sorting works
- [ ] Verify filtering works
- [ ] Verify pagination works
- [ ] Verify row selection works
- [ ] Verify empty state displays

### Navigation and Routing
- [ ] Verify navigation updates URL
- [ ] Verify back/forward buttons work
- [ ] Verify deep linking works
- [ ] Verify scroll position on navigation
- [ ] Verify browser history works

## Anti-Patterns

### Closing Without Verification
```tsx
// ❌ Incorrect: Closing issue without testing
// "I implemented the fix, closing the issue"
```

### Testing Only Happy Path
```tsx
// ❌ Incorrect: Only testing the main scenario
// "I tested it and it works"
```

### Skipping Edge Cases
```tsx
// ❌ Incorrect: Not testing edge cases
// "I tested the normal case, that should be enough"
```

### Ignoring Regressions
```tsx
// ❌ Incorrect: Not checking for regressions
// "I fixed the main issue, didn't check anything else"
```

## Correct Patterns

### Comprehensive Verification
```tsx
// ✅ Correct: Full verification checklist
// - Reproduced issue: Yes
// - Root cause: overflow: hidden on hero-container
// - Fix: Portal rendering implemented
// - Verified: Original issue resolved
// - Verified: No regressions in related components
// - Verified: Mobile behavior correct
// - Verified: Keyboard navigation works
// - Verified: Accessibility unaffected
```

### Documented Evidence
```tsx
// ✅ Correct: Document verification with evidence
// Verified fix in Chrome, Firefox, Safari
// Tested on mobile (375px), tablet (768px), desktop (1440px)
// Keyboard navigation: Tab, Enter, Escape all work
// No console errors or warnings
// No regressions in PropertyFilters, AgencyDirectoryClient
```

### PR Description Template
```tsx
// ✅ Correct: Include verification in PR
## Verification
- [x] Original issue resolved
- [x] No regressions observed
- [x] Mobile behavior verified
- [x] Keyboard navigation verified
- [x] Accessibility unaffected
- [x] Tested in Chrome, Firefox, Safari
```

## RealtorNet Requirements

### Minimum Verification
- [ ] Reproduced the issue
- [ ] Proven root cause
- [ ] Implemented fix
- [ ] Verified original issue resolved
- [ ] Verified no regressions

### Recommended Verification
- [ ] All minimum items
- [ ] Mobile behavior verified
- [ ] Keyboard navigation verified
- [ ] Browser refresh verified

### Full Verification
- [ ] All recommended items
- [ ] Accessibility unaffected
- [ ] Responsive layout verified
- [ ] State persistence verified
- [ ] Cross-browser testing

## Escalation Criteria

If any of these fail, escalate before closing:

- [ ] Original issue still reproducible
- [ ] New regressions introduced
- [ ] Keyboard navigation broken
- [ ] Mobile behavior broken
- [ ] Console errors or warnings
- [ ] Accessibility violations

## Documentation

Always document verification in:

1. PR description
2. Issue comments
3. Commit messages (if significant)
4. Code comments (if non-obvious)

Example PR description:
```
## Summary
Fixed hero search dropdown clipping by implementing portal rendering.

## Root Cause
Hero section has `overflow: hidden` which clipped the absolutely positioned dropdown.

## Fix
Implemented portal rendering using `createPortal` to render dropdown to `document.body`.

## Verification
- [x] Original issue resolved
- [x] No regressions observed
- [x] Mobile behavior verified (375px, 768px, 1440px)
- [x] Keyboard navigation verified (Tab, Enter, Escape)
- [x] Accessibility unaffected
- [x] Tested in Chrome, Firefox, Safari
- [x] Browser refresh verified
```
