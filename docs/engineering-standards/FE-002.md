# Standard FE-002 — Membership State Tab Routing Standard

This standard defines the canonical approach for tab routing in membership-related surfaces in RealtorNet.

## Principle

**Local state for immediate interaction, URL state for shareable navigation.**

Membership surfaces use local state management for tab switching with optional URL synchronization for bookmarkable states.

## Tab Structure

### Main Tabs (Level 1)
The primary navigation level for membership surfaces:

```tsx
type MembershipMainTab = "agencies" | "invitations" | "memberships" | "requests";
```

### Membership Sub-Tabs (Level 2)
Secondary navigation within the memberships tab:

```tsx
type MembershipSubTab = "active" | "rejected" | "suspended" | "left" | "revoked" | "blocked" | "history";
```

### Request Sub-Tabs (Level 2)
Secondary navigation within the requests tab:

```tsx
type RequestSubTab = "pending" | "rejected" | "cancelled";
```

### Invitation Sub-Tabs (Level 2)
Secondary navigation within the invitations tab:

```tsx
type InvitationSubTab = "pending" | "accepted" | "rejected";
```

## Implementation Pattern

### State Management

```tsx
export function MyJoinRequestsClient() {
  // Main tab state
  const [activeTab, setActiveTab] = useState<MembershipMainTab>("memberships");
  
  // Sub-tab states
  const [membershipSubTab, setMembershipSubTab] = useState<MembershipSubTab>("active");
  const [requestSubTab, setRequestSubTab] = useState<RequestSubTab>("pending");
  const [invitationSubTab, setInvitationSubTab] = useState<InvitationSubTab>("pending");
  
  // ... rest of component
}
```

### Tab Visibility Rules

Tabs should only be visible when they have data or are currently active:

```tsx
{[
  { value: "active", label: `Active (${activeMemberships.length})` },
  { value: "rejected", label: `Rejected (${requests.filter(r => r.status === "rejected").length})` },
  { value: "suspended", label: `Suspended (${suspendedMemberships.length})` },
  { value: "left", label: `Left (${leftMemberships.length})` },
  { value: "revoked", label: `Revoked (${revokedMemberships.length})` },
  { value: "blocked", label: `Blocked (${blockedMemberships.length})` },
  { value: "history", label: `History (${historyQuery.data?.length ?? 0})` },
].filter(t => {
  // Hide rejected tab if no rejected requests and not currently viewing
  if (t.value === "rejected") return requests.some(r => r.status === "rejected") || membershipSubTab === "rejected";
  // Hide history tab if no history and not currently viewing
  if (t.value === "history") return (historyQuery.data?.length ?? 0) > 0 || membershipSubTab === "history";
  // Always show other tabs
  return true;
}).map(({ value, label }) => (
  <Button 
    key={value} 
    type="button" 
    variant={membershipSubTab === value ? "primary" : "ghost"} 
    size="sm" 
    onClick={() => setMembershipSubTab(value)}
  >
    {label}
  </Button>
))}
```

### Tab Content Rendering

```tsx
{membershipSubTab === "active" ? (
  <ActiveMemberships memberships={activeMemberships} />
) : membershipSubTab === "rejected" ? (
  <RejectedRequests requests={requests.filter(r => r.status === "rejected")} />
) : membershipSubTab === "suspended" ? (
  <SuspendedMemberships memberships={suspendedMemberships} />
) : membershipSubTab === "left" ? (
  <LeftMemberships memberships={leftMemberships} />
) : membershipSubTab === "revoked" ? (
  <RevokedMemberships memberships={revokedMemberships} />
) : membershipSubTab === "blocked" ? (
  <BlockedMemberships memberships={blockedMemberships} />
) : membershipSubTab === "history" ? (
  <MembershipHistory history={historyQuery.data} />
) : null}
```

## Data Filtering

### Membership Status Filtering

```tsx
const activeMemberships = memberships.filter(m => m.status === "active");
const suspendedMemberships = memberships.filter(m => m.status === "suspended");
const leftMemberships = memberships.filter(m => m.status === "left");
const revokedMemberships = memberships.filter(m => m.status === "revoked");
const blockedMemberships = memberships.filter(m => m.status === "blocked");
```

### Request Status Filtering

```tsx
const pendingRequests = requests.filter(r => r.status === "pending");
const rejectedRequests = requests.filter(r => r.status === "rejected");
const cancelledRequests = requests.filter(r => r.status === "cancelled");
```

### Invitation Status Filtering

```tsx
const pendingInvitations = invitations.filter(i => i.status === "pending");
const acceptedInvitations = invitations.filter(i => i.status === "accepted");
const rejectedInvitations = invitations.filter(i => i.status === "rejected");
```

## Badge Variants

### Status Badge Mapping

```tsx
function getStatusVariant(status: string) {
  if (status === "approved" || status === "active") {
    return "success" as const;
  }
  if (status === "rejected" || status === "blocked" || status === "inactive" || status === "revoked") {
    return "danger" as const;
  }
  return "warning" as const;
}
```

### History Badge Mapping

```tsx
function getHistoryBadgeVariant(action: string) {
  if (action === "joined" || action === "reinstated") return "success" as const;
  if (action === "revoked" || action === "suspended" || action === "blocked") return "danger" as const;
  if (action === "left") return "warning" as const;
  return "outline" as const;
}
```

## Role-Based Access

### Tab Visibility by Role

```tsx
const token = getStoredToken();
const role = normalizeAppRole(getStoredJwtRole());

const canViewAgencyRequests = Boolean(token) && (role === "seeker" || role === "agent" || role === "agency_owner");
const canViewAgencyInvitations = Boolean(token) && (role === "seeker" || role === "agent");
const canViewAgencyMemberships = Boolean(token);
```

### Conditional Query Execution

```tsx
const requestsQuery = useMyAgencyJoinRequests(canViewAgencyRequests);
const membershipsQuery = useMyAgencyMemberships(canViewAgencyMemberships);
const historyQuery = useMembershipHistory(canViewAgencyMemberships);
const invitationsQuery = useMyAgencyInvitations(canViewAgencyInvitations);
```

## URL Synchronization (Optional)

For surfaces that require shareable URLs, synchronize tab state with URL query parameters:

```tsx
import { useSearchParams, useRouter } from 'next/navigation';

export function MyJoinRequestsClient() {
  const searchParams = useSearchParams();
  const router = useRouter();
  
  // Initialize from URL or default
  const [activeTab, setActiveTab] = useState<MembershipMainTab>(
    (searchParams.get('tab') as MembershipMainTab) || "memberships"
  );
  const [membershipSubTab, setMembershipSubTab] = useState<MembershipSubTab>(
    (searchParams.get('membershipTab') as MembershipSubTab) || "active"
  );
  
  // Update URL on tab change
  const handleTabChange = (tab: MembershipMainTab) => {
    setActiveTab(tab);
    const params = new URLSearchParams(searchParams);
    params.set('tab', tab);
    router.push(`?${params.toString()}`);
  };
  
  const handleMembershipSubTabChange = (subTab: MembershipSubTab) => {
    setMembershipSubTab(subTab);
    const params = new URLSearchParams(searchParams);
    params.set('membershipTab', subTab);
    router.push(`?${params.toString()}`);
  };
}
```

## Empty State Handling

Each tab should have an appropriate empty state:

```tsx
{activeMemberships.length === 0 ? (
  <div className="md:col-span-2 xl:col-span-3">
    <EmptyState 
      title="No active memberships" 
      description="You have no active agency memberships." 
    />
  </div>
) : (
  activeMemberships.map((membership) => (
    <MembershipCard key={membership.membership_id} membership={membership} />
  ))
)}
```

## Anti-Patterns

### Hardcoded Tab Count
```tsx
// ❌ Incorrect: Hardcoded count instead of dynamic
<Button>Active (3)</Button>
```

### Always Showing All Tabs
```tsx
// ❌ Incorrect: Showing empty tabs
{membershipSubTabs.map(({ value, label }) => (
  <Button key={value}>{label}</Button>
))}
```

### Missing Empty States
```tsx
// ❌ Incorrect: No empty state for empty tabs
{activeMemberships.map((membership) => (
  <Card key={membership.membership_id}>{membership.agency_name}</Card>
))}
```

## Correct Patterns

### Dynamic Counts
```tsx
// ✅ Correct: Dynamic count from data
<Button>Active ({activeMemberships.length})</Button>
```

### Conditional Tab Visibility
```tsx
// ✅ Correct: Hide empty tabs unless active
.filter(t => {
  if (t.value === "rejected") return requests.some(r => r.status === "rejected") || membershipSubTab === "rejected";
  return true;
})
```

### Empty State for Each Tab
```tsx
// ✅ Correct: Empty state for each tab
{activeMemberships.length === 0 ? (
  <EmptyState title="No active memberships" description="..." />
) : (
  activeMemberships.map((membership) => <Card key={membership.membership_id} />)
)}
```

## Migration Guide

When adding new membership-related surfaces:

1. Define tab types following the naming convention
2. Implement local state management with useState
3. Add dynamic counting for tab labels
4. Implement conditional tab visibility
5. Add empty states for each tab
6. Apply role-based access control
7. Use consistent badge variants
8. Optionally add URL synchronization for shareable states

## Testing

Verify tab routing by:

1. Switching between main tabs
2. Switching between sub-tabs
3. Verifying dynamic counts update
4. Verifying empty tabs hide unless active
5. Verifying empty states display correctly
6. Verifying role-based tab visibility
7. Testing URL synchronization if implemented
